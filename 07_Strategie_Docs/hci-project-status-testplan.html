4. Wat er nog moet gebeuren vóór go-live

✅ Fase A — Bouwen (klaar)

Alle content, pagina's, platform modules, briefs, analyses, blueprints, templates en tools zijn gebouwd.

🔶 Fase B — Review & Testen (NU)

Mike reviewt de deliverables lokaal. Checkt content op juistheid, pricing, positionering. Test interactieve elementen. Geeft feedback. Ik verwerk aanpassingen.

🔵 Fase C — Staging Deploy

Na goedkeuring: deploy naar staging URL (bijv. staging-hci.netlify.app). Testen in live-omgeving zonder DNS-wijziging. Netlify Identity, API keys, formulieren testen.

🔵 Fase D — Go-live

DNS switch naar productie. Alleen na succesvolle staging test. Geen herhaling van de vorige Netlify-disaster.

5. Testplan — hoe je lokaal kunt testen

5.1 P1 Website — lokaal openen

Alle P1 bestanden zijn standalone HTML. Je kunt ze direct in je browser openen.

01
Homepage openen

Open hci-homepage-wp1.html in je browser. Check: navigatie, drie route-links, Mike-sectie, bilingual toggle (NL/EN), mobile responsive (verklein je browservenster).

Dubbelklik op het bestand → opent in Chrome/Edge/Safari
02
Drie routes testen

Open hci-eu-entry-wp2.html, hci-growth-wp3.html, hci-validate-wp4.html. Check per route: hero copy, probleemomschrijving, 4-fasen journey, CTA's, case referenties.

Links tussen pagina's werken pas na deployment (relatieve paden)
03
Assessment Engine

Open hci-assessment-wp5.html. Doorloop de wizard. Check: vragen, scoring, resultaat-output, route-recommendation, email capture (FormSubmit).

FormSubmit werkt alleen op live domein, niet lokaal
04
Content Factory — Briefs

Open de 4 Public Briefs en 4 Opportunity Analyses. Check: cijfers kloppen, pillar-namen correct, pricing realistisch, CTA's werken.

Let op: logo (White_logo_-_no_background.png) moet in dezelfde map staan
05
GTM Calculator

Open gtm-calculator-growth.html. Test: toggle reguleringen aan/uit, wijzig klantaantal en maanden, check of scenario's correct berekenen, bar chart klopt.

Interactief — klik op de regulering-buttons en dropdowns
06
LinkedIn Templates

Open hci-linkedin-templates-wp6.html. Check: alle 40 posts aanwezig (10 per regulering), copy-to-clipboard werkt, content klopt.

Klik op de "Kopieer" knop bij een post → plak in kladblok
07
Layer 3 Blueprints (NIEUW)

Open layer3-nis2/dora/aiact-blueprint.html. Dit zijn jouw werkdocumenten. Review: PMC workshop klopt? Pricing realistisch? Competitive positioning herkenbaar? GTM-timeline haalbaar?

Elke blueprint heeft 📝 review-blokken met specifieke vragen

5.2 P2 Platform — lokaal testen (beperkt)

08
Theme & Navigation

Open hci-modules.html. Check: dark theme, HCI logo, navigation links, module badges met kleuren.

hci-nav.js moet in dezelfde map staan voor auto-detectie navigatie
09
Quickscan + Sectors

Open hci-quickscan.html. De Claude API-integratie werkt niet lokaal (vereist Netlify proxy). Maar de UI, vragen-flow en sector-selectie kun je wel testen.

AI-output = placeholder lokaal. Volledige test na staging deploy.
10
4-fase journey (PMC → GTM → Outreach → Sales)

Open hci-pmc.html, hci-gtm.html, hci-outreach.html, hci-sales-execute.html. Check: journey rail status, stappen-flow, formulieren, taal-toggle.

Journey rail auto-detectie werkt na deployment
⚠️ Wat NIET lokaal werkt en staging vereist:
• Claude API calls (quickscan, AI-analyse, output engine) — vereist ANTHROPIC_API_KEY via Netlify proxy
• Netlify Identity (login, registratie, invite) — vereist Netlify-omgeving
• FormSubmit email capture — vereist live domein
• Cross-page navigatie — vereist correct pad-structuur op server
• DOCX export met HCI branding — vereist serverside rendering

5.3 Staging Deploy — wanneer je klaar bent met lokale review

11
Staging URL opzetten

Deploy naar Netlify met staging URL (bijv. hci-staging.netlify.app). Geen DNS-wijziging. Environment variables instellen: ANTHROPIC_API_KEY. Netlify Identity activeren.

netlify deploy --dir=. (of via Netlify dashboard drag-and-drop)
12
End-to-end test op staging

Doorloop de volledige journey: homepage → route kiezen → assessment → resultaat → brief bekijken → Layer 2 downloaden → contact. Test ook: platform login → quickscan → sector output → DOCX export.

Dit is de finale test. Als dit werkt, is go-live een DNS switch.
