1. De Intelligence Flywheel — concept

Kernprincipe: elk contactpunt verrijkt het profiel

Het Intelligence Platform is geen losse tool. Het is een cirkel die zichzelf voedt: DealFlow verrijkt een contact → je stuurt een link → het platform trackt gedrag → gedrag verrijkt DealFlow → je volgende actie is slimmer → de klant ervaart een relevantere website → meer conversie → meer data → betere CMO→FMO output.

Het verschil met een gewone website: een gewone site is een folder. Het Intelligence Platform is een medewerker die onthoudt wie er langskwam, wat ze bekeken, en wat ze waarschijnlijk nodig hebben.

┌─────────────┐ ┌──────────────┐ ┌──────────────┐ ┌─────────────┐ │ DealFlow │───▶│ Outreach │───▶│ Website │───▶│ PostHog │ │ (profiel) │ │ (link+UTM) │ │ (herkenning) │ │ (gedrag) │ └──────▲──────┘ └──────────────┘ └──────────────┘ └──────┬──────┘ │ │ │ ┌──────────────┐ │ └──────────────│ Feedback │◀───────────────────────────┘ │ (verrijking)│ └──────────────┘ ◉ Flywheel: DealFlow → Outreach → Website → Gedrag → DealFlow → ...

2. Drie herkenningslagen

Geordend op kosten en betrouwbaarheid. Laag 1 en 2 vangen 70-80% van je bezoekers. Laag 3 is het vangnet.

L1
DealFlow → Persoonlijke tracking link (bekende contacten) — €0

Contact is verrijkt in DealFlow (KVK, Apollo, directors). Je stuurt een email of LinkedIn bericht met persoonlijke link: hci.com/?ref=deal_abc123 of hci.com/?p=jan-de-vries. Platform weet exact wie het is. Gedrag wordt teruggeschreven naar de deal-kaart. MEDDIC-signalen automatisch gevuld.

Wanneer: elke directe outreach vanuit DealFlow. Elke email. Elke LinkedIn InMail. Elke follow-up.

L2
LinkedIn campagne → UTM-segment routing (semi-bekend) — €0

Campagne-link bevat context: hci.com/?utm_segment=fintech&utm_campaign=dora_q2&utm_source=linkedin. Platform weet: segment = fintech, interesse = DORA, bron = LinkedIn. Laadt de juiste variant: DORA hero, fintech social proof, assessment CTA. Geen persoons-ID maar wel segment-ID.

Wanneer: LinkedIn Ads, sponsored posts, campagne-specifieke content, webinar registratie-links.

L3
IP-lookup → Bedrijfsherkenning (onbekende bezoekers) — €39-299/mnd

Bezoeker komt via Google, directe URL, of ongetagde link. Geen UTM, geen tracking-parameter. IP-adres → bedrijfsnaam, sector, omvang, land. Matcht aan ICP-segment. Laadt bijpassende variant. Als geen match: fallback naar ICP-quiz (2 vragen).

Wanneer: organisch verkeer (SEO), directe bezoeken, niet-getagde doorverwijzingen, onbekende LinkedIn klikken.

GDPR: Alleen bedrijfsniveau, geen persoonsidentificatie. Firmographic data = geen persoonsgegeven.

5. DealFlow integratie

5.1 Assessment → auto-lead

Wanneer iemand de assessment afrondt:

1. PostHog event assessment_completed met score, route, regulering.

2. Als email ingevuld → webhook naar DealFlow PocketBase API.

3. DealFlow createDeal() met: company (uit L1/L2/L3), contact (email), stage = "prospect", source = "Website Assessment", value (geschat op basis van regulering), meddic.identifyPain = true (assessment toont gap).

4. Deal-kaart bevat: assessment score, antwoorden, route, regulering, intent-signalen van PostHog.

5. Mike krijgt notificatie (email of DealFlow dashboard alert).

5.2 Persoonlijke tracking-link generatie

Vanuit DealFlow een link genereren:

1. In DealFlow: "Genereer outreach link" knop op deal-kaart.

2. Link wordt: hes-consultancy-international.com/?ref={deal_id}&r={regulation}&s={segment}

3. Wanneer contact klikt → Signal Layer leest ref → DealFlow lookup → profiel geladen → Personalization Layer past pagina aan.

4. Elke actie op de site wordt teruggeschreven naar die deal-kaart.

5.3 DealFlow schema-uitbreiding

Het huidige PocketBase schema ondersteunt dit grotendeels. Benodigde uitbreidingen:

9. Uitvoeringsplan (12 weken)

WEEK 1-2 Fundament

PostHog events implementeren. Alle core events (sectie 4.1) in de website code. Testen: klik door elke pagina, check of events binnenkomen in PostHog.

GA4 verificatie. Check: tracking code op alle pagina's, goals ingesteld voor assessment_completed en email_captured.

Snitcher account aanmaken. Script toevoegen aan site. Webhook naar DealFlow configureren (als beschikbaar) of handmatig checken.

PostHog dashboards bouwen. Funnel Performance + Recognition Coverage (sectie 4.3).

UTM-structuur vastleggen. Naming convention voor alle outbound links documenteren.

WEEK 3-4 DealFlow-koppeling

Assessment → auto-lead webhook. Assessment afgerond + email → POST naar DealFlow API. Deal aangemaakt met assessment data.

Tracking-link generator. Knop in DealFlow: "Genereer outreach link." Output: hci.com/?ref={deal_id}.

Signal Layer MVP. Vercel Edge Middleware die ref/utm/IP leest en profiel-object bouwt. Nog geen personalisatie — alleen tracking.

DealFlow schema uitbreiden. intent_signals, recognition_layer, tracking_link, regulation_interest, assessment_result (sectie 5.3).

WEEK 5-6 Personalisatie v1

Segment-based hero. 5 hero-varianten laden op basis van segment (ISV, Gemeente, Scale-up, Enterprise, Onbekend). UTM-segment en IP-match als input.

Regulering-pills dynamisch. DORA highlight voor fintech-bezoekers. NIS2 highlight voor overheid. CRA highlight voor software-bedrijven.

ICP-quiz fallback. 2-vragen quiz voor onbekende bezoekers. Resultaat → segment → personalisatie. PostHog event icp_quiz_completed.

WEEK 7-8 A/B testing launch

PostHog Feature Flags activeren. Eerste 2 experimenten live (homepage hero + CTA tekst).

Baseline meten. Eerste 2 weken data verzamelen zonder A/B (controle). Daarna split testen starten.

SEO technisch. Meta tags, structured data, sitemap, Google Search Console setup. Briefs als /insights/cra etc. publiceren.

WEEK 9-10 Feedback Loop sluiten

DealFlow intent-enrichment. Website gedrag (pages bekeken, scroll depth, assessment score) automatisch terugschrijven naar deal-kaart.

Content Effectiveness dashboard. Welke brief converteert het best? Welke regulering trekt het meest? Data-driven content prioritering.

Persoonlijke outreach links testen. 20 DealFlow-contacten een tracking-link sturen. Meten: open rate, klikgedrag, assessment start.

WEEK 11-12 Intelligence Flywheel compleet

Cross-site analytics. Als ChunkWorks dezelfde event-structuur draait: eerste cross-site learnings dashboard.

A/B test resultaten evalueren. Winnaars activeren. Nieuwe experimenten plannen (personalisatie impact, assessment lengte).

Flywheel review. Wat werkt? Wat niet? Welke herkenningslaag levert de meeste waarde? Snitcher ROI evalueren. Volgende kwartaal plannen.

10. Beslispunten voor Mike

📝 Beslispunt 1 — IP-lookup tool kiezen

Snitcher (NL, €39-79/mnd), Leadfeeder (€79-199/mnd), of Clearbit Reveal (€99-299/mnd)? Mijn advies: Snitcher — Nederlands, beste NL-bedrijfsdekking, laagste prijs, webhook support. Probeer de free trial eerst.

📝 Beslispunt 2 — UTM naming convention

Stel nu vast hoe je links tagt. Voorstel: utm_source (linkedin/email/website), utm_campaign (nis2_q2/dora_fintech), utm_segment (gemeente/fintech/isv/enterprise). Eenmaal gekozen, niet meer wijzigen.

📝 Beslispunt 3 — DealFlow tracking-link: automatisch of handmatig?

Optie A: "Genereer link" knop in DealFlow (bouw ik). Optie B: je maakt zelf links met een vaste structuur (hci.com/?ref=DEALID). Optie A is netter maar vereist DealFlow update. Optie B werkt morgen.

📝 Beslispunt 4 — Personalisatie: server-side of client-side?

Server-side (Vercel Edge): bezoeker ziet direct de juiste variant. Technisch complexer. Client-side (JavaScript swap): sneller te bouwen, milliseconden flash van origineel. Mijn advies: client-side voor nu, server-side als Fase 2.

📝 Beslispunt 5 — ChunkWorks alignment timing

Wil je Dennis nu al de event-structuur (sectie 4) sturen zodat hij dezelfde implementeert? Of wacht je tot je eigen site live is en getest? Mijn advies: stuur het nu — het kost Dennis niets en je kunt direct cross-site vergelijken.

📝 Beslispunt 6 — Assessment auto-lead: minimale data of volledig?

Optie A: alleen company + email + score → DealFlow. Optie B: volledige assessment data (antwoorden, route, regulering, intent-signalen). Mijn advies: Opt
