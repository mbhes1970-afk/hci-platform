Verwijderverzoek — vereiste flow
1

Verzoek ontvangen

Via e-mail aan mbhes@hes-consultancy-international.com of via zelfbedieningsknop in accountpagina. Identiteit verifieren via het e-mailadres waarmee geregistreerd.

Dag 0
2

Bevestiging sturen

Automatische ontvangstbevestiging met referentienummer en maximale behandelingstermijn (30 dagen conform AVG Art. 12).

Dag 0 — automatisch
3

Data zoeken en verwijderen

Alle records in PocketBase gefilterd op e-mailadres: CMO-FMO scans, quickscans, contacten, deals, activiteiten. Elk record verwijderen + loggen met timestamp en admin-ID.

Binnen 30 dagen
4

Bewijsdocument genereren

Geautomatiseerd document met: welke collections doorzocht zijn, hoeveel records verwijderd, timestamp, door wie uitgevoerd. Dit document bewaren in een aparte deletion_log collectie (geen persoonsdata, alleen metadata).

Gelijktijdig
5

Bevestiging naar betrokkene

E-mail met bevestiging van verwijdering, datum en beschrijving van welke categorieen zijn verwijderd. Geen persoonsdata meesturen in de bevestiging.

Uiterlijk dag 30

Opdracht voor Code

Maak een PocketBase collection deletion_log met velden: request_date, completion_date, collections_checked, records_deleted, executed_by, reference_number. Geen e-mailadres of naam opslaan in dit log — alleen aantallen en metadata. Maak een admin-script dat op basis van een e-mailadres alle gerelateerde records vindt, verwijdert en het log aanmaakt.

Sectie 04
Toestemmingsregistratie — CMO-FMO
Wat er juridisch vereist is bij het verzamelen van gegevens via de CMO-FMO wizard.

De CMO-FMO vraagt persoonsgegevens (naam, e-mail) voordat het rapport gegenereerd wordt. Dit vereist een geldige rechtsgrondslag. HCI kiest voor toestemming (AVG Art. 6 lid 1 sub a) omdat de verwerking niet strikt noodzakelijk is voor de dienst — het rapport kan technisch ook anoniem gegenereerd worden.

Vereiste checkbox teksten
Checkbox Tekst (verplicht exact zo) Verplicht?

Ik kan mijn toestemming te allen tijde intrekken via mbhes@hes-consultancy-international.com. Verplicht
Rapport delen met HCI Ik ga akkoord dat HES Consultancy International een kopie van mijn rapport ontvangt voor kwaliteits- en opvolgingsdoeleinden. Verplicht

Juridisch vereist — niet samenvoegen

De drie checkboxen mogen niet worden samengevoegd tot een checkbox. AVG Art. 7 lid 2 vereist dat toestemming per doel afzonderlijk wordt gevraagd. Bundelen maakt de toestemming ongeldig.

Toestemming opslaan in PocketBase
// PocketBase collection: cmo_fmo_consents { "id": "auto", "email": "contact@organisatie.nl", "org_name": "Gemeente Amsterdam", "consent_processing": true, // checkbox 1 — verplicht "consent_report_share": true, // checkbox 2 — verplicht "consent_followup": false, // checkbox 3 — optioneel "consent_timestamp": "2026-03-09T10:23:00Z", "ip_hash": "sha256_hash_not_plaintext", // gepseudonimiseerd "user_agent_hash": "sha256_hash", "withdrawn": false, // true als toestemming ingetrokken "withdrawn_date": null }

Implementatienoot voor Code

PocketBase ondersteunt gebruikersauthenticatie natively. Gebruik pb.collection('users').authWithPassword() voor Free/Pro. Voor CMO-FMO gebruikers: gebruik authWithOTP() (magic link) — geen wachtwoord vereist. Rol bepalen via een subscription_tier veld in de users collectie. Collection rules beperken op basis van @request.auth.subscription_tier.
